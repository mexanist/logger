# liblogger - динамическая библиотека для записи сообщений в журнал с разными уровнями важности

Поддеерживает многопоточную запись через потокобезопасную очередь

## Возможности

- Три уровня важности: ERROR, WARNING, INFO. Допустимы как строчные так и заглавные
  буквы в названиях уровнях важности, а так же цифровое отображение 0, 1, 2, где 0 наивысшая 
  степень важности, 2 - наименьшая.
- Сообщения с уровнем ниже заданного не записываются в журнал.
- После инициализации доступна возможность менять уровень важности сообщений.
- Формат записываемого сообщения: <дата> <уровень важности> <текст сообщения>

# logger_app - консольное приложение для проверки записи в журнал

Подключает и использует динамическую библиотеку для записи сообщений в журнал.
Принимает сообщение и уровень важности сообщения от пользователя. Уровень важности может 
отсутствовать. Передает принятые данные от пользователе в отдельном потоке для записи в журнал, 
передача данных потокобезопасна. Ожидает новый ввод от пользователя после передачи данных
Параметры приложения являются имя файла журнала и уровень важности сообщения по умолчанию:

```bash

./logger_app <файл_журнала> [уровень]

```
## Команды в интерактивном режиме

```
> текст сообщения [ERROR|WARNING|INFO]  — записать в журнал
> !level [уровень]                       — изменить/показать уровень по умолчанию
> !help                                  — показать справку
> !exit или Ctrl+C                       — выход

```

# Сборка

```bash

make all       # Собрать библиотеку и приложение
make library   # Только динамическая библиотека (.so)
make app       # Только консольное приложение
make run       # Собрать и запустить с параметрами по умолчанию
make clean     # Удаляет файлы результатов сборки, оставля исходные файлы

```

# Используемый инструментарий

- gcc, с поддержкой С++17
- make
- Linux kernel 6.1.0-50-686-pae (Debian 12 bookworm)
